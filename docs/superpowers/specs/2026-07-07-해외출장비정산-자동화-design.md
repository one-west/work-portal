# 해외출장비 정산 자동화 — 설계 문서

작성일: 2026-07-07
상태: 설계 확정 (구현 계획 대기)

## 1. 배경 / 목표

사내 통합 업무 포털(Streamlit 멀티페이지)에 **해외출장비 정산서 작성**을 자동화하는
새 페이지를 추가한다. 현재는 담당자가 카드 매입/승인 내역 엑셀에서 특정 카드의 사용
내역을 손으로 추출해 정산서 템플릿 `2-1출장경비 법인카드` 탭에 옮겨 적고, 환율 수식·
병합·합계·항목분류를 수작업으로 맞춘다. 이 과정을 앱에서 업로드→선택→편집→다운로드로
대체한다.

기준 검증: 이 정산 작업을 수동으로 1건(카드 `4074-6721-9739-3852`, 46건) 완주하여
로직을 확정했고, 독립 제작본("완성(코워크)")과 금액·구조가 100% 일치함을 확인했다.

## 2. 워크북 구조에서 나온 핵심 사실 (범위 결정 근거)

정산서 `.xlsx`는 이미 수식으로 연결된 시스템이다:

- **출장비 정산(요약) 탭 2개**(원화지급/외화지급): 각 항목 셀이
  `=SUMIFS(INDIRECT("'"&...))`로 detail 탭(2-1/2-2/2-3)에서 **자동 집계**. → 코드가
  건드릴 필요 없음. detail만 채우면 요약은 저절로 계산.
- **2-1 법인카드**: 전자 원천파일(해외매입내역, 국내승인내역)이 존재 → **유일하게 진짜
  자동 추출이 가능한 탭.**
- **2-2 현금 / 2-3 개인경비**: 원천이 종이 영수증 수기 입력. 자동 추출할 전자 데이터
  없음. → 앱은 관여하지 않음(향후 입력 폼은 별도 확장).
- **4.분류 시트**: 항목 마스터(13개) + 지출구분(법인카드/현금/개인경비). 분류의 정답 후보.
- **3개 detail 탭의 D열(항목)·E열(구분) 드롭다운은 x14 확장 데이터유효성** → openpyxl
  저장 시 소실(수용, §7).

**결론**: "정산서 전체 자동화"는 실질적으로 **`2-1` 자동 채움 + 상단 메타/환율 입력**으로
수렴한다. 나머지는 자동(요약) 또는 본질적 수기(현금/개인경비)다.

## 3. MVP 범위

- **포함**: 2-1 법인카드 detail 탭 자동 채움, 상단 출장 메타/환율표 입력, 항목 자동분류
  (앱에서 편집 가능한 규칙), 다중 카드 배치 생성.
- **자동으로 됨(코드 없음)**: 요약 탭 집계(템플릿 SUMIFS).
- **비목표(YAGNI)**: 현금/개인경비 자동추출·입력폼, 요약탭 코드작성, 규칙 외부저장, 학습
  캐시, 소분류(중식대/석식대 등) 자동분류, x14 드롭다운 복원.

## 6. 데이터 흐름

```
① 업로드: 해외매입.xls(선택) + 국내승인.xls(선택) [최소 1] + 카드마스터(매번)
        │  parsers.read_haewoe/read_gukne/read_card_master — xlrd, 카드번호 정규화
        ▼
② 카드 다중선택: 업로드 파일의 유니크 카드번호 → 카드마스터로 라벨링
        │  라벨 예: "해외출장4 · 김동현 · 미국 조지아"
        ▼
③ 카드별 처리(각 선택 카드):
        │  필터 → 날짜 오름차순 → 행 정규화(해외/국내 통합 스키마)
        │  classify(행, 원천): 해외=가맹점명 / 국내=업종명
        │  M열 = 카드마스터 출장자명(폴백: 이용자명/공용)
        ▼
④ 메타 입력표: 선택 카드별 출장자·기간·지역·목적·환율(USD/IDR)
        │  카드마스터에서 출장자·지역 프리필
        ▼
⑤ 편집: st.data_editor로 detail 행(D/M/F) 보정 + 규칙 편집(expander, JSON 다운/업)
        ▼
⑥ 생성: writer.build(template, card_rows, meta) — 카드별 1개 xlsx
        │  2-1: 행 9~(8+N) 확장, F:G·합계 병합, J/K 수식($L), L, 합계 SUM
        │  요약탭 상단: 출장자/기간/지역/목적 + 환율표 → SUMIFS 자동 집계
        ▼
⑦ 다운로드: 단일=xlsx, 다중=ZIP(카드별 "..._출장자명.xlsx")
```

### 통합 detail 행 스키마
`{date, shop, category(D), detail(F), usd(H), idr(I), krw(L), user(M), source}`
- 해외 건: usd=현지이용금액, idr=0, krw=결제원금
- 국내 건: usd=0, idr=0, krw=승인금액(원화)

## 7. 규칙 모델 (분류)

### JSON 스키마
```json
{
  "version": 1,
  "rules": [
    {"order": 1, "match": "contains", "keyword": "FOOD LION|COSTCO|WAL-MART|KROGER|TARGET",
     "category": "", "applies_to": "haewoe", "note": "마트 대량구매 유보"},
    {"order": 2, "match": "contains", "keyword": "WAWA|SHELL|EXXON|SPBU|GAS|TAXI|TOLL|PARKING",
     "category": "여비교통비", "applies_to": "common"},
    {"order": 8, "match": "regex", "keyword": "^[A-Z]{2,4}\\d{4,}$",
     "category": "", "applies_to": "haewoe", "note": "코드성 상호"}
  ]
}
```
- `order`: 작을수록 우선. `match`: `contains`(`|`구분 다중키워드) 또는 `regex`.
- `category`: `4.분류`의 13개 값 중 하나 또는 `""`(공란). `applies_to`: `haewoe`(가맹점명)/
  `gukne`(업종명)/`common`(둘 다).
- **classify(row, source)**: 대상 텍스트=(국내면 업종명, 해외면 가맹점명) → `applies_to ∈
  {source, common}` 규칙을 order순 검사 → 첫 매칭 category, 없으면 공란.

### 편집 UI / 검증
- 규칙 편집표(`st.data_editor`): 행 추가/수정/삭제/재정렬. `category`는 4.분류 값
  selectbox 컬럼으로 강제(오타 방지). 규칙 수정 시 즉시 재분류.
- JSON 다운로드/업로드로 개인이 규칙셋 보관·재적용(세션 한정, 재시작 시 기본셋 복귀).
- 기본셋 = 브레인스토밍에서 확정한 규칙표를 `rules_default.json`으로 직렬화.

### 자동화 불가 회색지대(문서화)
- **접대비 판정 불가**: 같은 상호가 맥락에 따라 식비/접대비. "고객 동석" 정보는 가맹점명에
  없음 → 기본 식비, 접대비 승격은 사람이 편집표에서.
- **마트 대량구매**: 식자재/물품 혼재 + `4.분류` 메모의 미결 정책($50 한도·팁·식자재) →
  공란 유지.
- **코드성 상호**(EOC03171 등): 공란.

## 8. 엑셀 Writer 상세

- **참조 스타일**: 데이터행(원본 11행)·합계행(원본 31행)의 셀 스타일/번호서식/행높이를
  수정 전 캡처해 복제.
- **행 확장**: 데이터 9~(8+N), 합계 9+N. 기존 예시(홍길동/2023 인니) 영역 클리어 후 기입.
- **병합**: 각 데이터행 `F:G`, 합계행 `A:C`+`F:G` 수동 재설정(openpyxl은 insert 시
  자동조정 안 함).
- **수식**: J=`=IF(H{r}>0,$L{r}/H{r},0)`, K=`=IF(I{r}>0,$L{r}/I{r},0)` (열 절대 `$L`,
  행 상대 — 원본 11행 형태로 전행 통일). 합계 H/I/L=`=SUM(...)`.
- **메타/환율**: 요약 탭 상단(출장자/기간/지역/목적) + 환율표(USD·IDR rate) 셀 기입 →
  2-2/2-3의 환율 참조와 요약 SUMIFS가 자동 계산.
- **템플릿 레이아웃 검증**: 7행 헤더·시트명 확인, 다르면 명확한 에러.
- **x14 드롭다운**: openpyxl 저장 시 소실(수용). category는 앱 편집표에서 통제.
  원본 템플릿은 리포에 보존.

## 9. 에러 처리 / 엣지 케이스

| 상황 | 처리 |
|---|---|
| 해외·국내 둘 다 미업로드 | 진행 차단 + 안내 |
| 선택 카드 0건 | 경고, 배치면 해당 카드 스킵(로그) |
| .xls 형식 오류(OLE2 아님) | 명확한 에러 메시지 |
| 취소/거절 건 | 국내 `상태`/`승인구분`, 해외 취소분 → 제외 필터 |
| 숫자 정규화 | 유니코드 공백·△▲ 음수·괄호 음수 처리(DART `to_number_strict` 패턴 재사용) |
| 카드 마스터 컬럼 상이 | 카드번호/출장자/지역 컬럼 자동 탐지, 실패 시 안내 |
| 카드가 마스터에 없음 | 라벨=원번호, M열=이용자명/공용 폴백 |
| 템플릿 레이아웃 불일치 | writer가 검증 후 에러 |
| 0 나눗셈 | J/K 수식 `IF(H>0,…)` 가드 |

## 10. 테스트 전략 (TDD)

- **parsers**: I/O와 순수 로직 분리 → 정규화·필터·정렬·숫자파싱·카드마스터 매핑을
  DataFrame 입력으로 단위 테스트.
- **rules**: classify(가맹점명 샘플→항목), order 우선순위, regex, 국내 업종명 분기, JSON
  라운드트립, category 검증.
- **writer**: 내장 템플릿에 채운 뒤 재로드해 N행·합계 SUM 범위·J/K `$L` 수식·병합·L값·
  메타셀 단언(수동검증 스크립트를 테스트로 승격).
- pytest, `tests/expense/`.

## 11. 리스크 / 미해결

- 카드 마스터 실제 파일 포맷(헤더/컬럼명)이 스크린샷 기준 추정 → 실제 파일로 파서 확정 필요.
- 국내승인(`4.승인내역조회`) 실데이터로 국내 경로(업종명 분류·승인금액·취소필터) 검증 필요
  (이번 카드는 국내 0건이라 미검증).
- x14 드롭다운 소실이 실제 운영에 불편하면 후속으로 zip 후처리 재주입 검토(갈림길 1-B).
